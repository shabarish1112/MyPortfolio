Library System

Efficient Database Management Solution

SQL Database Design Backend

Overview

The Library Management System is a comprehensive database solution designed to streamline the day-to-day operations of a library. It handles the complete lifecycle of book management, from acquisition and cataloging to member registration and circulation control. Built with a focus on data integrity and efficient retrieval, it ensures that library resources are managed effectively.

Key Features

  • Book Inventory: Detailed catalog with classification by genre, author, and publisher.
  • Member Management: Track member subscriptions, borrowing history, and contact details.
  • Circulation Control: Efficient check-in and check-out processes with due date tracking.
  • Fine Calculation: Automated calculation of late fees based on return dates.
  • Search & Retrieval: Optimized queries for fast lookup of books and availability status.

Technologies Used

Core database architecture implemented using SQL (MySQL/PostgreSQL) with a focus on normalization and relational integrity.

Schema Design (SQL)

The robust relational database structure powering the system.

CREATE DATABASE IF NOT EXISTS library_db;
USE library_db;

CREATE TABLE Book(
book_id INT PRIMARY KEY AUTO_INCREMENT,
title VARCHAR(100) NOT NULL,
author VARCHAR(100),
category VARCHAR(50),
status VARCHAR(20)DEFAULT 'Available'
);

CREATE TABLE Member (
    member_id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    phone VARCHAR(15)
);

CREATE TABLE Issue_Record (
    issue_id INT PRIMARY KEY AUTO_INCREMENT,
    book_id INT,
    member_id INT,
    issue_date DATE,
    return_date DATE,
    FOREIGN KEY (book_id) REFERENCES Book(book_id),
    FOREIGN KEY (member_id) REFERENCES Member(member_id)
);

CREATE TABLE Author (
    author_id INT PRIMARY KEY AUTO_INCREMENT,
    author_name VARCHAR(100) NOT NULL,
    nationality VARCHAR(50)
);

CREATE TABLE Publisher (
    publisher_id INT PRIMARY KEY AUTO_INCREMENT,
    publisher_name VARCHAR(100) NOT NULL,
    country VARCHAR(50)
);

CREATE TABLE Category (
    category_id INT PRIMARY KEY AUTO_INCREMENT,
    category_name VARCHAR(50) UNIQUE NOT NULL
);

-- (Sample Data Insertions Omitted for Brevity) --

CREATE TABLE Fine (
    fine_id INT PRIMARY KEY AUTO_INCREMENT,
    issue_id INT,
    amount DECIMAL(6,2),
    paid_status VARCHAR(10) DEFAULT 'Unpaid',
    FOREIGN KEY (issue_id) REFERENCES Issue_Record(issue_id)
);
